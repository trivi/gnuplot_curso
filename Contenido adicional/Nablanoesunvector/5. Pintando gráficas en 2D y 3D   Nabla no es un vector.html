5. Pintando gráficas en 2D y 3D

﻿
Jue, 10/07/2008 - 15:44 — grupo

Como se comentó en la introducción, con gnuplot podemos representar gráficas de funciones o representar un archivo de datos, tanto en 2D como en 3D, con muchísimas opciones de las cuales comentaremos algunas.

  • Pintar una función implícita
  • Definir tu propia función
  • Pintar más de una función en la misma gráfica
  • Representar los datos de un archivo
  • Pintar una lista de datos por consola
  • Especificar leyendas
  • Especificar estilo de representación (puntos, líneas, impulsos, barras, etc)
  • Color y forma de los puntos
  • Particularidades en la representación 3D
  • Replot
  • Ajuste de funciones

Gráficas 2D.
  • Pintar una función implícita (como las que traen las calculadoras), por ejemplo el arcoseno(x):

    plot acos(x)

    Incluye todas las funciones implícitas de uso común: trigonométricas (sin,cos,tan), hiperbólicas (sinh,cosh,tanh), logarítmicas (log,log10), exponencial (exp). Y las que no son de tan uso común pero pueden ser de utilidad a los físicos (funciones error, bessel, gamma, etc). Para ver el catálogo completo y sus especificaciones consultar la ayuda, dentro del programa ejecutar

    help functions

  • Definir tu propia función:
    se pueden definir constantes y funciones

    a=1
    g(x)=a*x**2+x**3+sin(x)

    Esta orden define la constante a=1 y la función g(x)=a*x^{2}+x^{3}+sin(x). Se pueden definir tantas funciones y constantes como se deseen.
    Por otra parte para una función definida a trozos se usa la estructura:

    y(x)=(x>=1) ? x : x**2

    Donde ha de leerse que si X es mayor o igual que la unidad entonces escribir la función x, en otro caso escribir la función x2, este es el resultado




  • Pintar más de una función en la misma gráfica:
    basta con separar cada función con comas

    plot sin(x), cos(x)

    Esta orden pintará en la misma gráfica el seno y el coseno, con diferentes leyendas.




  • Representar los datos de un archivo:

    plot 'datos.dat'

    La extensión .dat es arbitraria y la consola debe estar situada en el fichero donde esté el archivo.
    Si el archivo de datos solo tiene una columna representará cada dato frente a su cardinal (primer dato para x=1, segundo dato para x=2, etc).
    Si el archivo tiene dos columnas se representará la primera columna como la X y la segunda como la Y, sin embargo se puede especificar al revés:

    plot 'datos.dat' u 2:1

    Donde u 2:1 es la abreviatura de usando columna 2 para la X y columna 1 para la Y.
    Se puede jugar con este comando de infinidad de maneras, en especial vamos a especificar una construcción que usamos para que de un archivo con cuatro columnas
    X, Y, ΔX, ΔY
    obtengamos un gráfico con barras de error y cajas de error

    plot 'datos.dat' u 1:2:3:4 w boxxyerr, 'datos.dat' u 1:2:3:4 w xyerr

    La primera parte u 1:2:3:4 w boxxyerr pinta las cajas de error (cajas centradas sobre los puntos X,Y con lados 2*\Delta X y 2*\Delta Y) y la segunda parte u 1:2:3:4 w xyerr pinta las barras de errores. No entraremos en más discusión ya que las posibilidades son inmensas y esta es la configuración estándar para representar los datos del laboratorio.




  • Pintar una lista de datos por consola:
    esta opción no es muy útil por sí sola, pero nos servirá para crear animaciones. Si ingresamos

    plot '-'

    espera una lista de datos X Y (si solo introducimos un valor por línea, representa el valor frente a los cardinales, análogo al caso ANTERIOR).

    1 2
    2 3
    3 4.5
    e

    para terminar la lista hay que introducir e (end).




  • Especificar leyendas:
    para especificar las leyendas de una lista de datos usamos title 'titulo' o notitle de la siguiente manera

    plot 'datos1.dat' u 1:2 title 'datos1', 'datos2.dat' u 1:2 notitle

    Donde se pintará datos1.dat con la leyenda datos1 y se pintará datos2.dat sin leyenda.

  • Especificar estilo de representación (puntos, líneas, impulsos, barras, etc):
    para especificar si unimos los puntos con líneas o los dejamos como puntos, puntos minúsculos, color de los puntos, etc

    plot 'datos1.dat' u 1:2 w p

    Donde se pintará datos1.dat with points, con puntos. Entre otras opciones podemos especificar

    Código Resultado w l sólo con líneas w lp con puntos unidos por líneas w dots con puntos muy pequeños w i con impulsos, líneas de altura el valor determinado w boxes con barras (como un histograma), donde una tercera columna (u 1:2:3 o ingresando tres valores en plot '-')indica la anchura de la barra (centrada sobre el valor X).




  • Color y forma de los puntos:
    un número tras el estilo indica un color y forma distinto, así si no se pone nada

    plot 'datos1.dat' u 1:2 w p, 'datos2.dat' u 1:2 w p

    pinta los puntos de la primera con cruces + rojas y la segunda con cruces x verdes, pero

    plot 'datos1.dat' u 1:2 w p 0, 'datos1.dat' u 1:2 w p 5

    pinta la primera w p 0 con puntitos negros y la segunda w p 5 con cuadraditos turquesas, podéis probar distintos números.

Gráficas 3D.

Todo lo antedicho es válido para tres dimensiones pero con tres diferencias fundamentales:

  1. En vez del comando plot escribimos splot.
  2. Ahora las funciones pueden depender de X e Y, f(x,y).
  3. Las columnas de datos deben ser tres (el estilo boxes no se utiliza en 3D).

  • El comando replot es especialmente útil cuando queremos ver por consola el resultado de una gráfica y después meterlo en un archivo. En la sección de plantillas se explicará como usarlo.
  • Ajustes de funciones:
    Gnuplot te permite ajustar una determinada función que propones a los datos que le ofreces, dándote un margen de error. La manera de hacerlo es la siguiente, en el caso de un ajuste lineal

    y(x)=a*x+b
    fit y(x) 'datos1.dat' u 1:2 via a,b
    plot 'datos1.dat' u 1:2 title 'datos' , y(x) title 'ajuste'

    Donde y(x) es la función, cualquiera, que propones como ajuste (tened cuidado de no definir las constantes) y la segunda línea es el comando que lleva a cabo el ajuste ajustar 'datos1.dat' usando la columna 1 para la X y la columna 2 para la Y tomando como parámetros del ajuste las constante a y b.
    Cuando se ejecute saldrán los valores de a y b con sus respectivos errores en la consola.


    También hemos pintado la función y(x), el ajuste, para que aparezca en la gráfica.
